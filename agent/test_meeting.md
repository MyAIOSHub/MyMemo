# Q3 产品评审会议

[张总] 我们今天主要讨论 Q3 的功能优先级,先听听产品这边的想法
[李产品] 好的,我们准备了三个方案:智能推荐、用户画像、和社区功能
[王设计] 从设计角度,智能推荐的改版幅度最大,需要重做整个 Feed 页
[赵研发] 技术评估下来,推荐算法需要用到向量数据库,目前团队没有经验
[张总] 那推荐功能的投入产出比如何?有数据支撑吗?
[李产品] 根据竞品分析,推荐功能上线后 DAU 提升约 15%,但我们的用户基数不一样
[赵研发] 还有一个问题,向量数据库的运维成本不低,需要额外的基础设施预算
[王设计] 社区功能相对简单,但用户调研显示需求不强烈
[张总] 用户画像呢?这个和推荐是不是有关联?
[李产品] 是的,画像是推荐的基础,可以先做画像再做推荐,分两期
[赵研发] 画像的数据管道我们已经有了,改造成本低,大概两周就能上线
[张总] 那优先级就是:第一期做画像,第二期做推荐,社区先放一放?
[李产品] 对,但我觉得还需要定义清楚画像的北极星指标
[赵研发] 研发侧没问题,画像的 API 设计上周已经出了初稿
[王设计] 画像的展示页面我这边也有方案,需要过一下交互细节
[张总] 好,那就这样定了。下一步李产品出一个详细的 PRD,赵研发开始排期
